# Slice Reporting

## Overview

Adds a Reports screen to the Slice Planner app. Each completed slice gets a report page with description, test cases, test results, video recordings with interactive step-level trace navigation, and screenshots. Reports are generated by the "report" bead step and stored as artifacts that the Slice Planner app serves.

## Integration with Slice Planner

The Slice Planner app gains a third major view alongside Package View and Slice View:

- **Package View** — file-to-slice mapping (planning phase)
- **Slice View** — slice catalog and coverage (planning phase)
- **Reports View** — per-slice build reports (execution phase)

Navigation: slice list in the sidebar, clicking a slice shows its report. Slices without reports show a "pending" state.

## Report Content per Slice

1. **Header** — slice name, type (vertical/horizontal), description, status
2. **File list** — which files/packages are part of this slice
3. **Test cases** — the GWT (given/when/then) test designs from the test-design step
4. **Test results** — pass/fail per Maestro flow, against old app and new app side by side
5. **Video recordings** — embedded video player for each test run
6. **Interactive trace** — clickable step list synced to video timestamps
7. **Screenshots** — key screenshots captured during test flows

## Interactive Trace Component

### How it works
- Maestro's output includes step-level timing (step name, start time, duration)
- `maestro record` produces an mp4 synced to the test execution
- The trace UI renders steps as a clickable list alongside the video player
- Clicking a step calls `video.currentTime = stepStartSeconds` to seek
- Currently playing step is highlighted based on video playback position

### Data flow
```
maestro record → mp4 + results JSON (step timings)
                        ↓
              parse step names + timestamps
                        ↓
              render clickable step list + video player
```

### Offset calibration
There may be a small offset between Maestro's step timestamps and the video start. The report generator records the delta and the trace component applies it when seeking.

## Artifact Storage

```
reports/
  {slice-name}/
    report.json          # structured report data
    old_app/
      {flow-name}/
        video.mp4
        results.json     # maestro output with step timings
        screenshots/
          step-001.png
          step-002.png
    new_app/
      {flow-name}/
        video.mp4
        results.json
        screenshots/
          step-001.png
          step-002.png
```

The Slice Planner backend serves these from a configurable `reports/` directory.

## Report Generation Skill

### `slice-report` skill
- Runs as the final bead step (step 8) in the slice pipeline
- Collects artifacts from test-verify (step 5) and verify (step 7) output directories
- Builds `report.json` with:
  - Slice metadata (from slice YAML)
  - Test case definitions (from test-design output)
  - Test results summary (pass/fail per flow, old vs new)
  - Paths to videos and screenshots
  - Step timing data extracted from Maestro results JSON
- Stores everything in the `reports/{slice-name}/` directory
